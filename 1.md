Какие плюсы и минусы у REST?


| № | Плюсы                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  | Минусы                                                                                                                                                                                                                                 |
|---|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 1 | Простота. Можем использовать любой подход к взаимодействию в команде(contract first, code first). В случае с contract first описываем openapi схему, генерируем какие пустые интерфейсы, которые можем тут же замокать и получить готовый контракт (фронт и бэк могут начать работу одновременно). В случае с code first тоже есть плюсы, можем написать код и получить готовую схему буквально за одну секунду(большинство фреймфорков либо сами умеют в генерацию, либо есть сторонняя библиотека). В случае с code first тоже есть плюсы, можем написать код и получить готовую схему буквально за одну секунду(большенство фреймфорков либо сами умеют в генерацию, либо есть стороняя библеотека) | Надёжность. Никаких гарантий доставки и проблемы которые могут возникнуть при доставке                                                                                                                                                 |
| 2 | Распространенность. Слать рест запросы сегодня умеет буквально любая кофеварка.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        | Нет строгого стандарта ответов. Статус коды по-сути имеют рекомендательный характер и разработчик волен отвечать что угодно(кто-то на обработаный запрос будет кидать 200е, а кто-то 400е, встречал даже 500е при обработаном запросе) |
| 3 | Дешевизна. Вытекает из первых двух пунктов, любой бэкенд разработчик(даже уровня трейни) сможет отправить REST-запрос.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 | -                                                                                                                                                                                                                                      |

Какие плюсы и минусы у Messaging?


| Плюсы                                                   | Минусы                                                              |
|---------------------------------------------------------|---------------------------------------------------------------------|
| Надежности                                              | Понадобятся девопсы, для развёртывания брокера                      |
| Балансировка нагрузки                                   | Сложнее поддерживать, чем REST                                      |
| Разные типы гарантии доставки(в зависимости от брокера) | Ни один язык не поддерживает из коробки, нужны сторонние библиотеки |
| -                                                       | Нет лёгкого варианта описания контракта(как openapi)                |

Какие факторы должны учитываться при принятии решения? *(необязательный вопрос, ответьте по желанию)

1. Нет ресурсов или программистов, которые могут поддерживать взаимодействие через брокер сообщений - REST
2. Нет ресурсов или девопсов, для поддержания и равёртывания брокера сообщений - REST
3. Есть чёткое понимае что нужен моналит - REST
4. Если всё выше перечесленное не истина - Messaging

Какие плюсы и минусы у общей БД?


| Плюсы                                                                                           | Минусы                                                                                                                       |
|-------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------|
| Простота поддержки. Для поддержки нужно меньше людей, нет необходимости синхронизировать данные | Производительность будет ниже                                                                                                |
| Проще обновление                                                                                | Сложнее производить рефакторинг. База будет разростаться всё больше, добавление(обновление) полей таблицы становится сложнее |
| Стоймость будет ниже                                                                            | Усложнение дизайна системы, разграничение прав доступа к таблицам                                                            |
| -                                                                                               | Отказоустойчивость ниже. Падение базы данных ломает все приложение                                                           |

Какие плюсы и минусы у раздельных БД?


| Плюсы                                                                    | Минусы                                                                                         |
|--------------------------------------------------------------------------|------------------------------------------------------------------------------------------------|
| Отказоустойчивость выше. Падение базы данных не остановит все приложение | Нужно больше ресурсов и людей для поддержки                                                    |
| Проще производить рефакторинг. Базы меньше, в них проше разобраться      | Синхронизация данных(может возникать ситуация для разработчка, что данные будет сложнее найти) |
| Легче поддержка в рамках сервиса(команды)                                | Больше резервирования и накладных расходов(возможно потребуется больше DBA)                    |
| Производительность будет выше(меньше ссесий, меньше транзакций)          | Стоимость будет выше                                                                           |


Какие факторы должны учитываться при принятии решения? *(необязательный вопрос, ответьте по желанию)

1. Нет зависимых данных в разных сущностях (базы данных не зависят друг от друга) - Раздельная
2. Важна производительность - Раздельная
